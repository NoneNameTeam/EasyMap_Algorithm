## 数据结构说明 (Data Structure)

接口输入数据采用 JSON 格式，包含图的节点概况、坐标信息以及边权数据。

### 字段详解

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `n` | Integer | **节点数量**。 |
| `start` | Integer | **起点**节点 ID。 |
| `target` | Integer | **终点**节点 ID。 |
| `coords` | Array | **坐标列表**。<br>后端决定坐标，用于预估两个节点的相对距离。<br>**注意**：列表顺序固定，索引对应从节点 1 到 n。 |
| └ `x` | Number | 节点的 X 坐标。 |
| └ `y` | Number | 节点的 Y 坐标。 |
| `edges` | Array | **边集（邻接表）**。<br>描述图的连接关系及权重。 |
| └ `u` | Integer | 边的**出发**节点 ID。 |
| └ `v` | Integer | 边的**到达**节点 ID。 |
| └ `w` | Number | 边的**代价** (Weight)。<br>数值可代表路程、时间等量度。 |

### 示例数据

```json
{
  "n": 3,
  "start": 1,
  "target": 2,
  "coords": [
    { "x": 1, "y": 1 },
    { "x": 2, "y": 2 },
    { "x": 3, "y": 3 }
  ],
  "edges": [
    { "u": 1, "v": 2, "w": 1.5 },
    { "u": 2, "v": 3, "w": 1.5 }
  ]
}
